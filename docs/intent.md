# Intent - 意图层

## 1. 为什么做这个项目？
### 背景与动机
- 需要一个简约，游戏风格的游戏笔记内容网站
- 用于记录自己的游戏攻略、心得，以免时间过长遗忘

### 期望价值
- 能够展示图文内容，视频链接

## 2. 目标用户
### 主要用户：发布者
- 角色：内容创作者
- 核心需求：便捷地发布和管理博客内容
- 使用场景：通过编写Markdown文件来创作内容

### 次要用户：读者
- 角色：普通访客
- 核心需求：获取有价值的技术内容
- 使用场景：通过搜索引擎直接访问

### 说明
发布者也可以是读者，回看自己的游戏历程

## 3. 验收标准
### 必须达成（Must Have）
- [ ] 发布者能通过撰写 .md 文件发布内容
- [ ] 读者访问首页即可获取最新内容
- [ ] 高度 SEO 优化
- [ ] 支持多种游戏分类（单游戏下又可以分其他类别，比如攻略，Build，地图）

### 期望达成（Should Have）
- [ ] 良好的阅读体验
- [ ] 响应式设计

### 可选达成（Nice to Have）
- [ ] 文章搜索功能
- [ ] RSS 订阅

## 4. 明确不做什么
- 不做用户登录/注册系统
- 不做评论系统（可考虑第三方集成）
- 不做后台管理界面
- 不做多语言支持
